### 1. HashMap的底层实现原理

HashMap的底层实现原理是什么？JDK7和JDK8有什么区别？

**原理分析**

**JDK7 vs JDK8 区别：**

| 特性 | JDK7 | JDK8 |
|-----|------|------|
| 数据结构 | 数组+链表 | 数组+链表+**红黑树** |
| 哈希冲突 | 链表 | 链表→红黑树（链表长度≥**8**） |
| 插入方式 | 头插法 | 尾插法 |
| 扩容 | 重新hash | 原位置或原位置+旧容量 |
| 扰动 | 2次 | 1次 |

**JDK8 HashMap结构：**

```java
transient Node<K,V>[] table;

static class Node<K,V> implements Map.Entry<K,V> {
    final int hash;
    final K key;
    V value;
    Node<K,V> next;
    TreeNode<K,V> parent, left, right;
    boolean red;
}
```

**put流程：**

```java
public V put(K key, V value) {
    return putVal(hash(key), key, value, false, true);
}

final V putVal(int hash, K key, V value, boolean onlyIfAbsent, boolean evict) {
    // 1. 初始化
    if ((tab = table) == null || (n = tab.length) == 0)
        n = (tab = resize()).length;
    // 2. 计算位置
    if ((p = tab[i = (n - 1) & hash]) == null)
        tab[i] = newNode(hash, key, value, null);
    else {
        // 3. key相同，覆盖
        if (p.hash == hash && ((k = p.key) == key || (k != null && key.equals(k))))
            e = p;
        // 4. 红黑树插入
        else if (p instanceof TreeNode)
            e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
        // 5. 链表插入
        else {
            for (int binCount = 0; ; ++binCount) {
                if ((e = p.next) == null) {
                    p.next = newNode(hash, key, value, null);
                    if (binCount >= TREEIFY_THRESHOLD - 1)
                        treeifyBin(tab, hash);
                    break;
                }
            }
        }
    }
    if (++size > threshold) resize();
}
```

> HashMap为什么容量是2的幂？

```java
// 计算位置
index = (n - 1) & hash
// n是2的幂时，(n-1)的二进制全是1
// 等价于 hash % n，但位运算更快
```

### 2. ConcurrentHashMap的原理与JDK8的优化

ConcurrentHashMap的原理是什么？JDK7和JDK8有什么区别？

**原理分析**

**JDK7 vs JDK8 区别：**

| 特性 | JDK7 | JDK8 |
|-----|------|------|
| 数据结构 | Segment数组+HashEntry数组+链表 | **Node数组**+链表+红黑树 |
| 并发粒度 | Segment（继承ReentrantLock） | **Node**（CAS+synchronized） |
| 锁数量 | 16个Segment | 数组长度 |
| 锁方式 | 继承ReentrantLock的Segment | **synchronized + CAS** |

**JDK8 ConcurrentHashMap：**

```java
transient volatile Node<K,V>[] table;

static class Node<K,V> implements Map.Entry<K,V> {
    final int hash;
    final K key;
    volatile V value;
    volatile Node<K,V> next;
}
```

**put流程：**

```java
final V putVal(K key, V value, boolean onlyIfAbsent) {
    if (key == null || value == null) throw new NullPointerException();
    int hash = spread(key.hashCode());
    for (Node<K,V>[] tab = table;;) {
        // 1. 初始化
        if (tab == null || (n = tab.length) == 0)
            tab = initTable();
        // 2. 空位，CAS插入
        else if ((f = tabAt(tab, i = (n - 1) & hash)) == null) {
            if (casTabAt(tab, i, null, new Node<K,V>(hash, key, value, null)))
                break;
        }
        // 3. 扩容协助
        else if ((fh = f.hash) == MOVED)
            tab = helpTransfer(tab, f);
        // 4. 同步插入
        else {
            synchronized (f) {
                // 链表/红黑树插入
            }
        }
    }
}
```

> ConcurrentHashMap为什么放弃Segment？

1. **内存占用**：Segment继承ReentrantLock，有额外开销
2. **复杂度**：分段锁需要两次哈希定位
3. **粒度**：JDK8使用更细粒度的synchronized+CAS，性能更好

### 3. HashMap的线程安全问题与解决方案

HashMap是线程安全的吗？多线程环境下使用HashMap会有什么问题？

**原理分析**

**线程不安全原因：**

1. **并发put导致数据丢失**：

```java
// 线程A: 计算hash=5，准备插入
// 线程B: 计算hash=5，准备插入
// 线程A: put成功
// 线程B: put成功，覆盖A的值
```

2. **resize时出现死循环**（JDK7头插法）：

```java
// JDK7 resize
void transfer(Entry[] newTable) {
    for (Entry<K,V> e : src) {
        do {
            Entry<K,V> next = e.next;
            e.next = newTable[i];
            newTable[i] = e;
            e = next;
        } while (e != null);
    }
}
// 并发resize导致环形链表，get时死循环
```

3. **可见性问题**：

```java
HashMap map = new HashMap();
map.put("key", "value");
// 线程B读取可能为null
```

**解决方案：**

```java
// 1. ConcurrentHashMap
ConcurrentHashMap<Integer, Integer> map = new ConcurrentHashMap<>();

// 2. Collections.synchronizedMap
Map<Integer, Integer> map = Collections.synchronizedMap(new HashMap<>());

// 3. Hashtable（不推荐，性能差）
Hashtable<Integer, Integer> map = new Hashtable<>();
```

> Collections.synchronizedMap的实现原理？

所有方法都使用synchronized包裹同一个mutex对象。

### 4. HashMap的扩容机制与阈值计算

HashMap的扩容机制是怎样的？扩容阈值如何计算？

**原理分析**

**触发条件：**

1. 元素数量 > **threshold**（capacity * loadFactor）
2. 链表长度 >= TREEIFY_THRESHOLD（8），且数组长度 < 64

**默认参数：**

```java
static final int DEFAULT_INITIAL_CAPACITY = 1 << 4;    // 16
static final float DEFAULT_LOAD_FACTOR = 0.75f;
static final int TREEIFY_THRESHOLD = 8;
static final int MIN_TREEIFY_CAPACITY = 64;
```

**threshold计算：**

```java
this.threshold = tableSizeFor(initialCapacity);

// tableSizeFor：返回>=cap的最小2的幂
static final int tableSizeFor(int cap) {
    int n = -1 >>> Integer.numberOfLeadingZeros(cap - 1);
    return (n < 0) ? 1 : (n >= MAXIMUM_CAPACITY) ? MAXIMUM_CAPACITY : n + 1;
}
```

**扩容流程：**

```java
final Node<K,V>[] resize() {
    Node<K,V>[] oldTab = table;
    int oldCap = oldTab == null ? 0 : oldTab.length;
    int oldThr = threshold;
    int newCap, newThr = 0;

    if (oldCap > 0) {
        if (oldCap >= MAXIMUM_CAPACITY) {
            threshold = Integer.MAX_VALUE;
            return oldTab;
        }
        else if ((newCap = oldCap << 1) < MAXIMUM_CAPACITY)
            newThr = oldThr << 1;
    }
    // 迁移元素：原位置或原位置+旧容量
    // e.hash & oldCap 判断是否需要移动
}
```

> 负载因子为什么是0.75？

- **0.5**：扩容频繁，空间利用率低
- **0.75**：平衡点，扩容后约75%利用率
- **1.0**：空间利用率高，但冲突增多，性能下降

### 5. ArrayList与LinkedList的区别与选择

ArrayList和LinkedList有什么区别？各自的适用场景是什么？

**原理分析**

**区别对比：**

| 特性 | ArrayList | LinkedList |
|-----|----------|------------|
| 底层结构 | 动态数组 | 双向链表 |
| 随机访问 | O(1)（下标） | O(n)（需遍历） |
| 头部插入/删除 | O(n)（移动元素） | **O(1)** |
| 尾部插入/删除 | 均摊O(1) | O(1) |
| 空间占用 | 连续内存 | 每个节点有额外指针开销 |
| 内存局部性 | 好 | 差 |

**选择建议：**

```java
// 随机访问为主 → ArrayList
for (int i = 0; i < list.size(); i++) {
    String s = list.get(i);  // ArrayList快
}

// 头部/中间插入删除为主 → LinkedList
list.add(0, "a");  // LinkedList快
```

> ArrayList的扩容机制？

扩容为**1.5倍**：
```java
int newCapacity = oldCapacity + (oldCapacity >> 1);
```

### 6. ConcurrentHashMap的get方法无锁原理

ConcurrentHashMap的get方法为什么不需要加锁？它是如何保证可见性的？

**原理分析**

**无锁get原理：**

```java
public V get(Object key) {
    Node<K,V>[] tab; Node<K,V> e, p; int n, i, eh;
    int h = spread(key.hashCode());
    if ((tab = table) != null && (n = tab.length) > 0 &&
        (e = tabAt(tab, i = (n - 1) & h)) != null) {
        if ((eh = e.hash) == h) {
            if ((ek = e.key) == key || (ek != null && key.equals(ek)))
                return e.value;
        }
        else if (eh < 0)
            return (p = e.find(h, key)) != null ? p.value : null;
        while ((e = e.next) != null) {
            if (e.hash == h && ((ek = e.key) == key || (ek != null && key.equals(ek))))
                return e.value;
        }
    }
    return null;
}
```

**可见性保证：**

1. **volatile value**：

```java
static class Node<K,V> {
    volatile V value;  // volatile保证可见性
}
```

2. **tabAt使用getObjectVolatile**：

```java
static final <K,V> Node<K,V> tabAt(Node<K,V>[] tab, int i) {
    return (Node<K,V>)U.getObjectVolatile(tab, ...);
}
```

> ConcurrentHashMap的get和put可以同时进行吗？

可以：
- get读的是已经存在的节点
- put如果该位置为null，CAS写入
- 如果该位置有节点，synchronized锁住该节点进行修改

### 7. 红黑树的特性与HashMap中的实现

红黑树有哪些特性？HashMap中的红黑树是如何实现的？

**原理分析**

**红黑树特性：**

1. 节点是红色或黑色
2. 根节点是黑色
3. 叶子节点（NIL）是黑色
4. **红节点的子节点必须是黑色**
5. **任意节点到每个叶子节点的路径包含相同数量的黑节点**

**平衡操作：**

```java
// 左旋
static <K,V> TreeNode<K,V> rotateLeft(TreeNode<K,V> root, TreeNode<K,V> p) {
    TreeNode<K,V> r = p.right;
    p.right = r.left;
    if (r.left != null) r.left.parent = p;
    r.parent = p.parent;
    if (p.parent == null) root = r;
    else if (p.parent.left == p) p.parent.left = r;
    else p.parent.right = r;
    r.left = p;
    p.parent = r;
    return root;
}
```

**插入调整：**

每次插入后需要**重新着色和旋转**来保持平衡。

> 红黑树与AVL树的区别？

| 特性 | 红黑树 | AVL树 |
|-----|-------|------|
| 平衡度 | 黑高平衡 | 严格平衡（左右高度差≤1） |
| 插入/删除开销 | O(1)旋转 | O(logN)旋转 |
| 查询性能 | 略差于AVL | 最优 |

### 8. CopyOnWriteArrayList的原理与适用场景

CopyOnWriteArrayList的原理是什么？它适合什么场景？

**原理分析**

**核心思想：**

写操作时复制整个数组，读取不需要加锁。

```java
private transient volatile Object[] array;

// 读操作
public E get(int index) {
    return getAt(array, index);
}

// 写操作：加锁+复制
public boolean add(E e) {
    synchronized (lock) {
        Object[] elements = getArray();
        Object[] newElements = Arrays.copyOf(elements, len + 1);
        newElements[len] = e;
        setArray(newElements);
    }
}
```

**特性：**

| 特性 | 说明 |
|-----|------|
| 读写分离 | 读无锁，写加锁 |
| 最终一致性 | 写操作后读可能读到旧数据 |
| 内存占用 | 每次写都复制，内存占用高 |
| 适用场景 | **读多写少**（配置、缓存） |

> CopyOnWriteArrayList的迭代器特性？

迭代器不会抛出**ConcurrentModificationException**，因为迭代器遍历的是创建迭代器时的数组快照。

### 9. HashMap的hash方法扰动设计

HashMap的hash方法是如何实现的？为什么要这样设计？

**原理分析**

**JDK8 hash方法：**

```java
static final int hash(Object key) {
    int h;
    return (key == null) ? 0 :
        (h = key.hashCode()) ^ (h >>> 16);
}
```

**为什么这样做：**

1. **扰动**：高16位与低16位异或，让hash更分散

```
原始hash: 11111111 11111111 11111111 11111111
高16位:   11111111 11111111
低16位:   11111111 11111111
异或后:   00000000 00000000 11111111 11111111
```

2. **与运算优化**：

```java
// n是2的幂，hash & (n-1) 等价于 hash % n
// 扰动后分散了高16位信息
```

3. **解决负数问题**：

```java
index = (n - 1) & hash;
// 结果始终在 [0, n-1] 范围内
```

> 扰动为什么用异或而不是与/或？

**异或的特性**：
- 相同为0，不同为1
- 可逆：a ^ b ^ b = a
- 混合高位和低位信息，让结果更分散

### 10. ConcurrentHashMap的多线程扩容机制

ConcurrentHashMap是如何进行扩容的？多线程如何协助扩容？

**原理分析**

**触发条件：**

```java
// 元素数量 > 阈值
while (s >= (long)(sc = sizeCtl) && (tab = table) != null) {
    if (tab.length < MAXIMUM_CAPACITY)
        transfer(tab, nextTab);
}
```

**transfer流程：**

```java
private final void transfer(Node<K,V>[] tab, Node<K,V>[] nextTab) {
    int n = tab.length, stride;
    // 步长：每个线程处理至少16个桶
    int size = (n >>> 3) / 16;

    // 第一个线程创建新数组
    if (nextTab == null) {
        nextTab = new Node[n << 1];
    }

    for (int i = 0; i < n; i++) {
        for (Node<K,V> f = tabAt(tab, i); f != null; ) {
            if (f.hash == MOVED) {
                // 已经是ForwardingNode，帮助迁移
                tab = helpTransfer(tab, f);
            } else {
                synchronized (f) {
                    // 迁移单个桶
                }
            }
        }
    }
}
```

**ForwardingNode：**

```java
static final class ForwardingNode<K,V> extends Node<K,V> {
    final Node<K,V>[] nextTable;
    // find方法：去新数组查找
}
```

> 多线程如何协助扩容？

```java
// 线程A在put时发现正在扩容
else if ((fh = f.hash) == MOVED)
    tab = helpTransfer(tab, f);

final Node<K,V>[] helpTransfer(Node<K,V>[] tab, Node<K,V> f) {
    Node<K,V>[] nextTab = f.nextTable;
    if (nextTab != null) transfer(tab, nextTab);
    return nextTab;
}
```

帮助迁移的线程会参与扩容工作，加速扩容完成。

### 11. HashMap为什么容量必须是2的幂次方？

HashMap为什么要求容量必须是2的幂次方？这样做的好处是什么？

**原理分析**

**计算索引的方式：**

```java
index = (n - 1) & hash
```

当n为2的幂次方时，`n-1`的二进制形式全是1：
- n = 16 (10000)，n-1 = 15 (01111)
- n = 32 (100000)，n-1 = 31 (011111)

**好处：**

1. **位运算代替取模**：
   ```java
   hash % n  →  (n-1) & hash
   ```
   位运算比取模快很多倍

2. **分布均匀**：
   ```
   n-1 = 01111 (15)
   hash = 10010110 (150)
   150 & 15 = 6  (分布均匀)
   ```

3. **扩容时便于迁移**：
   ```java
   // 只需判断hash的第n位
   newIndex = (hash & oldCap) == 0 ? oldIndex : oldIndex + oldCap
   ```
   只需判断hash的特定位是0还是1，决定放在原位置还是原位置+oldCap

**如果不是2的幂次方：**

```java
// n = 15 (01111)，capacity = 15
hash = 10010110 (150)
150 & 14 = 6  // 0110
// 但15的二进制是0111，少了最高位的1
// 导致某些位置永远无法命中，分布不均匀
```

> 如果手动指定容量不是2的幂次方会怎样？

```java
new HashMap<>(10);  // 实际容量是16
new HashMap<>(15);  // 实际容量是16
new HashMap<>(17);  // 实际容量是32
```

HashMap会自动调整为最近的2的幂次方：`tableSizeFor(17) = 32`

### 12. HashMap遍历方式有哪些？哪种最优？

HashMap有哪些遍历方式？各自的特点是什么？

**原理分析**

**4种遍历方式：**

```java
Map<Integer, String> map = new HashMap<>();
for (int i = 0; i < 100; i++) map.put(i, "value" + i);

// 1. keySet遍历（不推荐）
for (Integer key : map.keySet()) {
    String value = map.get(key);  // 每次get需要再计算hash
}

// 2. entrySet遍历（推荐）
for (Map.Entry<Integer, String> entry : map.entrySet()) {
    Integer key = entry.getKey();
    String value = entry.getValue();
}

// 3. Iterator遍历
Iterator<Map.Entry<Integer, String>> it = map.entrySet().iterator();
while (it.hasNext()) {
    Map.Entry<Integer, String> entry = it.next();
}

// 4. Lambda遍历（JDK8+）
map.forEach((k, v) -> System.out.println(k + "=" + v));
```

**性能对比：**

| 方式 | 时间复杂度 | 额外开销 | 推荐 |
|-----|-----------|---------|------|
| keySet + get | O(n*k) | 每次重新计算hash | 不推荐 |
| entrySet | O(n) | 一次遍历 | **推荐** |
| Iterator | O(n) | 迭代器对象 | 推荐 |
| forEach | O(n) | Lambda开销 | 小数据量推荐 |

**最佳实践：**

```java
// 推荐：使用entrySet
for (Map.Entry<K, V> entry : map.entrySet()) {
    K key = entry.getKey();
    V value = entry.getValue();
}
```

> 为什么keySet不推荐？

因为`map.get(key)`需要再次计算hash并查找，额外开销大：

```java
public V get(Object key) {
    Node<K,V> e;
    return (e = getNode(hash(key), key)) == null ? null : e.value;
}
```

### 13. Hashtable与ConcurrentHashMap的区别

Hashtable与ConcurrentHashMap有什么区别？为什么推荐使用ConcurrentHashMap？

**原理分析**

**区别对比：**

| 特性 | Hashtable | ConcurrentHashMap |
|-----|-----------|------------------|
| 底层实现 | 数组+链表 | 数组+链表+红黑树（JDK8） |
| 锁粒度 | 整个表（synchronized） | 桶（synchronized + CAS） |
| 并发度 | 单锁 | 高（多个桶可并发） |
| 扩容 | 一次性扩容 | 多线程协助扩容 |
| 迭代器 | 强一致性 | 弱一致性 |
| null支持 | key/value都不允许为null | 都不允许为null |

**Hashtable实现：**

```java
public synchronized V get(Object key) { ... }
public synchronized V put(K key, V value) { ... }
public synchronized V remove(Object key) { ... }
// 所有操作都需要获取整个表的锁
```

**ConcurrentHashMap实现：**

```java
// JDK8
public V get(Object key) {
    Node<K,V>[] tab; Node<K,V> e, p; int n, i, h;
    // 无锁读，利用volatile保证可见性
}

public V put(K key, V value) {
    // CAS + synchronized实现
    // 只锁定单个桶，其他桶可以并发操作
}
```

**性能对比：**

```java
// 1000个线程并发写入
Hashtable:      ~5000ms
ConcurrentHashMap: ~200ms  // 25倍性能提升
```

**迭代器区别：**

```java
// Hashtable：强一致性，遍历期间不允许修改
// ConcurrentHashMap：弱一致性，遍历期间可以修改
```

> 为什么ConcurrentHashMap不允许null？

为了避免歧义：
- null.get() 返回null，是"不存在"还是"值就是null"？
- 统一禁止null，避免歧义

### 14. HashSet的底层实现

HashSet的底层实现是什么？它与HashMap有什么区别？

**原理分析**

**HashSet实现：**

```java
public class HashSet<E> extends AbstractSet<E> implements Set<E>, Cloneable, Serializable {
    private transient HashMap<E, Object> map;
    private static final Object PRESENT = new Object();  // 统一的value

    public HashSet() {
        map = new HashMap<>();
    }

    public boolean add(E e) {
        return map.put(e, PRESENT) == null;
    }

    public boolean contains(Object o) {
        return map.containsKey(o);
    }

    public boolean remove(Object o) {
        return map.remove(o) != null;
    }
}
```

**实际上HashSet内部是一个HashMap**，所有元素作为key，value使用一个统一的空对象PRESENT。

**HashSet vs HashMap：**

| 特性 | HashSet | HashMap |
|-----|---------|---------|
| 底层 | HashMap（value统一为PRESENT） | 数组+链表+红黑树 |
| 唯一性 | 保证元素不重复 | key唯一 |
| 允许null | 允许一个null | 允许一个null key |
| 线程安全 | 不安全 | 不安全 |

**LinkedHashSet：**

```java
// 维护一个双向链表，保持插入顺序
public class LinkedHashSet<E> extends HashSet<E> implements Set<E> {
    // 继承HashSet，使用LinkedHashMap
}
```

**TreeSet：**

```java
// 基于TreeMap实现，内部是红黑树
public class TreeSet<E> extends AbstractSet<E> implements NavigableSet<E> {
    private transient NavigableMap<E,Object> m;
    // 底层是TreeMap
}
```

> HashSet如何保证元素唯一性？

利用HashMap的key唯一性：
```java
public boolean add(E e) {
    return map.put(e, PRESENT) == null;  // null表示新插入
}
```

### 15. TreeMap的底层实现与排序

TreeMap的底层实现是什么？它如何保证key的有序性？

**原理分析**

**TreeMap实现：**

```java
public class TreeMap<K,V> extends AbstractMap<K,V> implements NavigableMap<K,V> {
    private final Comparator<? super K> comparator;
    private transient Entry<K,V> root;

    static final class Entry<K,V> implements Map.Entry<K,V> {
        K key;
        V value;
        Entry<K,V> left;
        Entry<K,V> right;
        Entry<K,V> parent;
        boolean color = BLACK;
    }
}
```

**底层是红黑树**，与HashMap中的TreeNode相同。

**put操作：**

```java
public V put(K key, V value) {
    Entry<K,V> t = root;
    if (t == null) {
        compare(key, key);  // 类型检查
        root = new Entry<>(key, value, null);
        size = 1;
        return null;
    }
    int cmp;
    Entry<K,V> parent;
    // 按Comparator比较查找位置
    do {
        parent = t;
        cmp = compare(key, t.key);
        if (cmp < 0) t = t.left;
        else if (cmp > 0) t = t.right;
        else return setValue(value);
    } while (t != null);
    Entry<K,V> e = new Entry<>(key, value, parent);
    if (cmp < 0) parent.left = e;
    else parent.right = e;
    fixAfterInsertion(e);  // 保持红黑树平衡
    return null;
}
```

**与HashMap的区别：**

| 特性 | TreeMap | HashMap |
|-----|---------|---------|
| 底层 | 红黑树 | 数组+链表+红黑树 |
| 顺序 | 有序（按key排序） | 无序 |
| 时间复杂度 | O(logN) | O(1)平均 |
| 插入/查找 | O(logN) | O(1)平均 |
| 适用场景 | 需要排序 | 需要快速查找 |

**Comparator vs Comparable：**

```java
// 使用Comparator
TreeMap<Person, String> map = new TreeMap<>(new Comparator<Person>() {
    @Override
    public int compare(Person p1, Person p2) {
        return p1.age - p2.age;  // 按年龄排序
    }
});

// 使用Comparable（让Person实现Comparable）
class Person implements Comparable<Person> {
    int age;
    @Override
    public int compareTo(Person p) {
        return this.age - p.age;
    }
}
```

> TreeMap的 ceilingKey、floorKey 方法？

```java
// 大于等于key的最小key
map.ceilingKey(key);

// 小于等于key的最大key
map.floorKey(key);
```

### 16. LinkedHashMap的原理

LinkedHashMap的原理是什么？它与HashMap有什么区别？

**原理分析**

**LinkedHashMap实现：**

```java
public class LinkedHashMap<K,V> extends HashMap<K,V> {
    // 双向链表头结点
    transient LinkedHashMap.Entry<K,V> head;
    transient LinkedHashMap.Entry<K,V> tail;

    // 节点继承HashMap.Node，增加了前后指针
    static class Entry<K,V> extends HashMap.Node<K,V> {
        LinkedHashMap.Entry<K,V> before, after;
        Entry(int hash, K key, V value, Node<K,V> next) {
            super(hash, key, value, next);
        }
    }
}
```

**继承HashMap**，但内部维护一个**双向链表**来保持插入顺序。

**两种顺序：**

```java
// 插入顺序（默认）
LinkedHashMap<Integer, String> map = new LinkedHashMap<>();
map.put(1, "a");
map.put(2, "b");
map.put(3, "c");
// 遍历：1->a, 2->b, 3->c

// 访问顺序（LRU缓存）
LinkedHashMap<Integer, String> map = new LinkedHashMap<>(16, 0.75f, true);
map.get(1);  // 访问后移到末尾
```

**LRU缓存实现：**

```java
class LRUCache extends LinkedHashMap<Integer, String> {
    private final int capacity;

    public LRUCache(int capacity) {
        super(16, 0.75f, true);
        this.capacity = capacity;
    }

    @Override
    protected boolean removeEldestEntry(Map.Entry<Integer, String> eldest) {
        return size() > capacity;  // 超过容量删除最老的
    }
}
```

**与HashMap的区别：**

| 特性 | LinkedHashMap | HashMap |
|-----|--------------|---------|
| 顺序 | 维护双向链表 | 无序 |
| 内存 | 额外指针开销 | 较小 |
| 遍历 | 按插入/访问顺序 | 随机 |
| 适用场景 | LRU缓存、保持顺序 | 快速查找 |

> LinkedHashMap如何实现LRU？

重写`removeEldestEntry`方法：
```java
@Override
protected boolean removeEldestEntry(Map.Entry eldest) {
    return size() > MAX_SIZE;
}
```

### 17. ConcurrentHashMap的size方法如何实现？

ConcurrentHashMap的size方法是如何实现的？为什么不能直接用变量累加？

**原理分析**

**size方法实现：**

```java
public int size() {
    long n = sumCount();
    return (n < 0) ? 0 : (n > Integer.MAX_VALUE) ? Integer.MAX_VALUE : (int)n;
}

final long sumCount() {
    CounterCell[] cells = counterCells;
    long sum = baseCount;
    if (cells != null) {
        for (CounterCell cell : cells) {
            if (cell != null) {
                sum += cell.value;  // 累加所有cell的值
            }
        }
    }
    return sum;
}
```

**为什么不用变量直接累加？**

```java
// 错误的实现
private int size = 0;
public void add() {

}
```

```java
static final class CounterCell {
```

```java
            !(uncontended = a.cas(v = a.value, v + check))) {

        }

}
```

### 18. Map遍历时如何安全地删除元素？

如何在遍历Map时安全地删除元素？

**原理分析**

**错误方式：**

```java
// ConcurrentModificationException
for (Map.Entry<Integer, String> entry : map.entrySet()) {
    if (entry.getValue().equals("delete")) {
        map.remove(entry.getKey());  // 会抛异常
    }
}

// 迭代过程中修改
Iterator<Map.Entry<Integer, String>> it = map.entrySet().iterator();
while (it.hasNext()) {
    Map.Entry<Integer, String> entry = it.next();
    if (entry.getValue().equals("delete")) {
        map.remove(entry.getKey());  // 会抛异常
    }
}
```

**正确方式1：使用Iterator.remove()**

```java
Iterator<Map.Entry<Integer, String>> it = map.entrySet().iterator();
while (it.hasNext()) {
    Map.Entry<Integer, String> entry = it.next();
    if (entry.getValue().equals("delete")) {
        it.remove();  // 使用迭代器的remove
    }
}
```

**正确方式2：使用removeIf（JDK8+）**

```java
map.entrySet().removeIf(entry -> entry.getValue().equals("delete"));
```

**正确方式3：HashMap使用keySet.remove()**

```java
for (Integer key : new ArrayList<>(map.keySet())) {
    if (map.get(key).equals("delete")) {
        map.remove(key);  // 先复制key再删除
    }
}
```

**正确方式4：ConcurrentHashMap安全遍历**

```java
// ConcurrentHashMap可以安全在遍历时删除
for (Map.Entry<Integer, String> entry : map.entrySet()) {
    if (entry.getValue().equals("delete")) {
        map.remove(entry.getKey());  // 不会抛异常
    }
}
```

**原理：**

- HashMap：遍历时modCount会变化，检测到则抛异常
- Iterator.remove()：先修改modCount，再从链表删除
- ConcurrentHashMap：使用弱一致的迭代器，不检测modCount

> Iterator.remove()的原理？

```java
public void remove() {
    if (lastRet < 0) throw new IllegalStateException();
    synchronized (expectedModCount) {
        Node<K,V> p = current;
        // 从hashmap中删除节点
        unlinkNode(p);
        // 同步迭代器的expectedModCount
        expectedModCount = modCount;
    }
}
```
