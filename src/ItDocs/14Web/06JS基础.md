## 1. Symbol是什么？

Symbol 是 ES6 引入的第 7 种数据类型，表示唯一、不可变的值。任何两个 Symbol 都不相等（== 或 === 都不相等），即使描述字符串相同。

- **局部 Symbol**: `Symbol("description")`，每次调用都创建一个不同的值
- **全局 Symbol**: `Symbol.for("description")`，全局注册表中复用，相同描述返回相同 Symbol

传入的描述字符串仅用于调试说明，不是值本身。建议始终传入描述参数以便调试和代码可读性。

参考：[理解ES6中的Symbol](https://blog.csdn.net/lihongxun945/article/details/48554117)

## 2. call、apply、bind 的作用是什么？

三者都用于改变函数执行时的 `this` 指向：

- **bind()**: 返回一个绑定了指定 `this` 的新函数，不立即执行。不论怎么调用都有同样的 `this` 值
- **call()**: 立即执行，参数逐个传入
- **apply()**: 立即执行，参数以数组传入

bind 最常用场景：将方法从对象中取出作为回调时，绑定原对象的 `this`，避免丢失原来的对象。

```js
var module = { x: 81, getX: function() { return this.x; } };
var retrieveX = module.getX;
retrieveX(); // 9，this 指向全局
var boundGetX = retrieveX.bind(module);
boundGetX(); // 81
```

参考：[bind MDN](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Function/bind)

## 3. ES6 class 是什么？

class 是 ES6 引入的语法糖，底层仍是基于原型链的继承，写法更接近传统面向对象语言。

```js
class Animal {
  constructor(name) { this.name = name; }
  speak() { console.log(this.name); }
}
class Dog extends Animal {
  speak() { console.log(this.name + ' barks'); }
}
```

## 4. this 的指向规则是什么？

`this` 在函数调用时确定，指向调用函数的对象：

1. **默认绑定**: 独立函数调用，非严格模式指向全局对象（window/global），严格模式为 `undefined`
2. **隐式绑定**: 作为对象方法调用，指向该对象
3. **显式绑定**: 通过 `call`/`apply`/`bind` 指定
4. **new 绑定**: `new` 调用时指向新创建的实例
5. **箭头函数**: 无自己的 `this`，继承定义时外层作用域的 `this`，固定不变

优先级：new > 显式绑定 > 隐式绑定 > 默认绑定

## 5. JavaScript 的作用域是什么？

作用域决定了变量和函数的可访问范围。

- **全局作用域**: 代码任何地方都可访问
- **函数作用域**（var）: 函数内声明的变量只在函数内可访问
- **块级作用域**（let/const）: `{}` 内声明的变量只在块内可访问
- **作用域链**: 访问变量时从当前作用域逐级向上查找，直到全局

ES6 之前只有全局作用域和函数作用域，`let`/`const` 引入了块级作用域。

参考：[深入理解JavaScript作用域](https://juejin.im/post/6844903584891420679)

## 6. 什么是闭包？

闭包是指能访问外部函数内部变量的函数，本质上是一个函数内部定义的函数。在 JavaScript 中，只有函数内部的子函数才能读取局部变量，所以闭包是将函数内部与外部连接起来的桥梁。

适用场景：
- **数据私有化**: 隐藏变量，只暴露特定接口
- **维持变量**: 让变量常驻内存，不被垃圾回收
- **回调/定时器**: 保留上下文数据

```js
function outer() {
  let count = 0;
  return function inner() {
    count++;
    return count;
  };
}
```

注意：闭包会导致变量不释放，过度使用可能引发内存泄漏。

参考：[学习Javascript闭包](https://www.cnblogs.com/hongru/archive/2010/10/30/1865041.html)

## 7. 箭头函数与 function 的区别？

1. **this 指向不同**: `function` 的 `this` 随调用环境变化；箭头函数没有自己的 `this`，继承定义时的外层 `this`，固定不变
2. **不能用作构造函数**: 箭头函数不能 `new`
3. **没有 arguments 对象**: 箭头函数使用剩余参数 `...args` 代替
4. **不能用作 generator**: 箭头函数不能使用 `yield`
5. **语法更简洁**: 单表达式可省略 `return`

参考：[箭头函数与普通函数的区别](https://blog.csdn.net/github_38851471/article/details/79446722)

## 8. let 和 const 与 var 的区别？

| 特性 | var | let | const |
|------|-----|-----|-------|
| 作用域 | 函数级 | 块级 | 块级 |
| 变量提升 | 有（undefined） | 有（暂存死区，不可访问） | 有（暂存死区，不可访问） |
| 重复声明 | 允许 | 不允许 | 不允许 |
| 重新赋值 | 允许 | 允许 | 不允许 |
| 初始值 | 可选 | 可选 | 必须 |

- **暂存死区（TDZ）**: `let`/`const` 声明前访问变量会报 ReferenceError
- **const** 保证的是引用不可变，对象内部属性仍可修改

参考：[let和const](https://juejin.im/post/6844903741892591630)

## 9. ES6 的 import 和 export 如何使用？

ES6 模块化使用 `export` 导出、`import` 导入：

**导出**：
```js
// 命名导出
export const name = 'value';
export function fn() {}
// 默认导出
export default class Person {}
```

**导入**：
```js
import { name, fn } from './module'; // 命名导入
import Person from './module';        // 默认导入
import * as all from './module';      // 全部导入
```

特性：
- 静态分析，编译时确定依赖关系
- 支持按需加载（`import()` 动态导入）
- 导出的是值的引用（只读），不是拷贝

参考：[ES6模块化](https://juejin.im/post/6844903733243936782)

## 10. JavaScript 创建对象及添加属性的方式？

**创建对象**：
1. **对象字面量**: `const obj = {}`
2. **构造函数**: `new Object()` / 自定义构造函数
3. **Object.create()**: `Object.create(proto)`
4. **class**: `new class Foo {}`

**添加/访问属性**：
1. **点语法**: `obj.key = value`
2. **方括号**: `obj['key'] = value`（支持变量和特殊字符）
3. **Object.defineProperty()**: 定义带属性描述符的属性（writable、enumerable、configurable）
4. **Object.defineProperties()**: 批量定义

参考：[MDN 使用对象](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Guide/Working_with_Objects)
