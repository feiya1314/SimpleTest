## 1. 什么是立即执行函数（IIFE）？有什么作用？

IIFE（Immediately Invoked Function Expression）是定义后立即执行的函数表达式。任何库与框架设计的第一个要点就是解决命名空间与变量污染的问题，jQuery 就是利用 JavaScript 函数作用域的特性，采用 IIFE 包裹自身方法来解决这个问题。

```js
(function(window, undefined) {
  var jQuery = function() {};
  // ...
  window.jQuery = window.$ = jQuery;
})(window);
```

jQuery 初始化时将自身加入 window 全局变量，外部可直接用 `$` 或 `jQuery` 调用其方法。

**优势**：
- **隔离作用域**：内部变量不会污染全局
- **减少作用域查找**：将 window 作为参数传入，闭包内作为局部变量使用，比在全局作用域查找更快
- **undefined 安全**：undefined 在 JS 中不是保留字，可被重新赋值；IIFE 不传递第二个参数，确保内部 undefined 是真正的 `undefined`

jQuery 等库利用此模式，只暴露 `$` / `jQuery` 到全局，内部实现完全隔离，自动初始化只构建一次。

## 2. 什么是全局对象？如何在不同环境中访问？

全局对象是 JS 运行环境提供的根对象，所有全局变量都是其属性。

- **浏览器**：`window`，也等同于 `self`、`frames`
- **Web Worker**：`self`
- **Node.js**：`global`
- **通用方案（ES2020）**：`globalThis`，在任何环境中均指向全局对象

```js
console.log(globalThis === window);  // 浏览器中为 true
console.log(globalThis === global);  // Node.js 中为 true
```

## 3. requestAnimationFrame 是什么？

`requestAnimationFrame` 是浏览器提供的动画 API，告诉浏览器在下一次重绘前执行指定回调。

**与 setTimeout/setInterval 对比**：

| 特性 | requestAnimationFrame | setTimeout/setInterval |
|------|----------------------|----------------------|
| 执行时机 | 与浏览器刷新同步（通常 60fps） | 固定延迟，可能掉帧 |
| 资源消耗 | 页面不可见时自动暂停 | 持续执行，浪费资源 |
| 精度 | 高，与 vsync 同步 | 低，受事件队列阻塞影响 |
| 调用方式 | 递归调用，无间隔参数 | 指定固定间隔 |

**适用场景**：CSS 无法实现的复杂动画（如 canvas 逐帧绘制、进度条平滑更新）。

```js
function animate() {
  // 更新动画逻辑
  requestAnimationFrame(animate);
}
requestAnimationFrame(animate);
```

## 4. 页面加载时 JS 脚本如何执行？

**浏览器解析 HTML 的流程**：

1. 解析 HTML 构建 DOM 树，遇到 `<script>` 会**暂停解析**，下载并执行 JS 后再继续（脚本阻塞特性）
2. CSSOM 构建也会阻塞 JS 执行（JS 可能操作样式）

**脚本加载方式**：

| 方式 | 行为 | 执行顺序 |
|------|------|---------|
| 普通 `<script>` | 遇到即阻塞，下载并立即执行 | 按文档顺序 |
| `<script defer>` | 与 HTML 解析并行下载，解析完成后执行 | 按文档顺序，在 `DOMContentLoaded` 前 |
| `<script async>` | 与 HTML 解析并行下载，下载完成后立即执行 | 无关文档顺序，谁先下载完谁先执行 |

**关键事件**：
- **DOMContentLoaded**：HTML 解析完成，DOM 树就绪（外部资源如图片可能未加载完）
- **load**：页面所有资源（图片、样式等）全部加载完成

**推荐实践**：将 `<script>` 放在 `</body>` 前，或使用 `defer` 属性，避免阻塞渲染。
