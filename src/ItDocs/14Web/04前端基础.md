## 1. Ajax原理是什么？

Ajax（Asynchronous JavaScript and XML）通过在后台与服务器进行少量数据交换，实现网页的异步更新，不重新加载整个页面即可局部刷新。

核心步骤：
1. 创建XMLHttpRequest对象
2. 设置请求参数（open方法）
3. 发送请求（send方法）
4. 监听状态变化（onreadystatechange）
5. 处理服务器响应

参考：[Ajax原理一篇就够了](https://www.zhihu.com/search?type=content&q=Ajax%E5%8E%9F%E7%90%86)

## 2. 前后端分离的原理是什么？

前后端分离的核心是前端通过Ajax/Fetch等技术与后端进行数据交互，前后端各自独立开发部署，通过HTTP API接口进行通信。

- **前端**：负责UI展示、路由控制、状态管理，部署在Nginx/CDN
- **后端**：提供RESTful/GraphQL API，只处理业务逻辑和数据
- **优点**：职责清晰、开发效率高、可独立扩展、多端共用接口

参考：[你不得不了解的前后端分离原理](https://juejin.im/post/5b71302351882560ea4afbb8)

## 3. CSS中块级元素、行内元素、行内块元素有什么区别？

每个元素生成了一个包含内容的框（box），但不同元素显示方式不同。

**块级元素（block）**：独自占一行，可设置宽高。
- 独自占一行，默认宽度为父元素100%
- 高度、行高、margin、padding都可控制
- 默认从上至下排列
- 常见：div、p、h1~h6、ul、ol、li、table、form、header、footer、section、article、nav

**行内元素（inline）**：不可设置宽高，与其他行内元素位于一行。
- 不会独占一行，一行排不下自动换行，宽度由内容决定
- 宽高设置无效，margin/padding仅左右方向有效，上下无效
- 行内元素中不能放块级元素
- 常见：span、a、strong、em、b、i、label、code、br

**行内块级元素（inline-block）**：可设置宽高，也可一行多个。
- 高度、行高、margin、padding都可控制
- 默认宽度为内容宽度，不独占一行
- 之间有空白缝隙，设置父级font-size为0可消除
- 常见：button、input、textarea、select、img

**元素类型转换（display）**：
- `display: block` — 块级元素
- `display: inline` — 行内元素
- `display: inline-block` — 行内块级元素

**总结**：区别在于排列方式、宽高边距设置、默认宽度。

## 4. Flexbox布局是什么？

Flexbox布局也叫弹性盒子布局。设为Flex布局后，子元素的`float`、`clear`和`vertical-align`属性将失效。

## 5. CSS padding、margin的区别，top和margin-top的区别

**padding（内边距）**：内容到边框的距离，不允许负值。
- `padding: 10px 5px;` — 上下10px，左右5px
- `padding: 1px 2px 3px 4px;` — 上右下左
- `padding: 10px;` — 全部10px

**margin（外边距）**：边框到外部元素的距离，允许负值。
- `margin: 10px 5px 15px 20px;` — 上右下左
- `margin: 10px 5px;` — 上下10px，左右5px
- `margin: 10px;` — 全部10px

**top与margin-top的区别**：
1. top/bottom/left/right是**绝对定位**，必须设置`position: absolute`才生效；margin是**相对定位**
2. top在绝对定位下相对body或`position: relative`的父级元素；margin相对相邻元素定位

## 6. Vue如何实现组件化和模块化？

Vue通过webpack实现模块化，使用`import`引入模块。一个组件就是一个独立模块，包含模板（template）、样式（style）和逻辑（script）等。

## 7. npm run dev的Vue本地路由如何工作？

`npm run dev`的Vue本地路由使用Node.js**反向代理**HTTP请求，类似于nginx，主要用于本地开发调试。生产环境反向代理仍使用nginx等反向代理服务。
