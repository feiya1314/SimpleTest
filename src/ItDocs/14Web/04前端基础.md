## 1. Ajax原理是什么？

Ajax（Asynchronous JavaScript and XML）通过在后台与服务器进行少量数据交换，实现网页的异步更新，不重新加载整个页面即可局部刷新。

核心步骤：
1. 创建**XMLHttpRequest**对象
2. 设置请求参数（open方法）
3. 发送请求（send方法）
4. 监听状态变化（onreadystatechange）
5. 处理服务器响应

参考：[Ajax原理一篇就够了](https://www.zhihu.com/search?type=content&q=Ajax%E5%8E%9F%E7%90%86)

## 2. 前后端分离的原理是什么？

前后端分离的核心是前端通过Ajax/Fetch等技术与后端进行数据交互，前后端各自独立开发部署，通过HTTP API接口进行通信。

- **前端**：负责UI展示、路由控制、状态管理，部署在Nginx/CDN
- **后端**：提供RESTful/GraphQL API，只处理业务逻辑和数据
- **优点**：职责清晰、开发效率高、可独立扩展、多端共用接口

## 3. CSS中块级元素、行内元素、行内块元素有什么区别？

每个CSS元素生成了一个包含内容的框（box），不同元素显示方式不同，区别在于排列方式、宽高边距设置、默认宽度。

**块级元素（block）**：独自占一行，可自定义宽度和高度，可容纳块级元素和行内元素。
- 每个块级元素独自占一行
- 高度、行高、margin、padding都可控制
- 默认宽度为父元素100%
- 多个块级元素默认从上至下排列
- 块级元素标签：div、p、h1~h6、ul、ol、li、table、form、address、caption、dd、dl、dt、fieldset、hr、legend、noframes、noscript、pre、tbody、td、tfoot、th、thead、tr、header、footer、article、section、aside、nav、figcaption、figure

**行内元素（inline）**：不可设置宽高，与其他行内元素位于一行，高度由字体大小决定，宽度由内容长度控制，一般不可包含块级元素。
- 不会独占一行，相邻行内元素排列在同一行，一行排不下自动换行
- 宽高设置无效，margin/padding仅左右方向有效，上下无效
- 设置行高有效，等同于给父级设置行高
- 宽度由内容决定
- 行内元素中不能放块级元素，a链接内不能再放链接
- 行内元素标签：span、a、strong、em、b、i、label、code、br、abbr、acronym、bdo、big、cite、dfn、kbd、q、samp、select、small、sub、sup、textarea、tt、var

**行内块级元素（inline-block）**：兼具块级和行内特点，可设置宽高，也可一行多个。
- 高度、行高、margin、padding都可控制
- 默认宽度为内容宽度，不独占一行
- 之间有空白缝隙，设置父级font-size为0可消除
- 行内块级元素：button、input、textarea、select、img

**元素类型转换 display**：
- `display: block` — 块级元素
- `display: inline` — 行内元素
- `display: inline-block` — 行内块级元素

**总结**：
- 块级元素独占一行，内联元素和内联块元素在一行内显示
- 块级元素和内联块元素可设置width/height，内联元素设置无效
- 块级元素width默认100%，内联元素由内容或子元素决定宽度

## 4. Flexbox布局是什么？

Flexbox布局也叫弹性盒子布局。设为Flex布局以后，子元素的**float**、**clear**和**vertical-align**属性将失效。

## 5. CSS中padding、margin、top有什么区别？

**padding（内边距）**：内容到边框的距离，不允许负值。
- `padding: 10px 5px;` — 上下10px，左右5px
- `padding: 1px 2px 3px 4px;` — 上右下左
- `padding: 10px;` — 所有4个内边距都是10px

**margin（外边距）**：边框到外部元素的距离，允许负值。
- `margin: 10px 5px 15px 20px;` — 上右下左
- `margin: 10px 5px;` — 上下10px，左右5px
- `margin: 10px;` — 所有4个外边距都是10px

**top与margin-top的区别**：
1. top/bottom/left/right是**绝对定位**，必须设置`position: absolute`才生效；margin是**相对定位**
2. top在绝对定位下相对body或`position: relative`的父级元素定位；margin相对相邻元素定位

## 7. npm run dev的Vue本地路由如何工作？

`npm run dev`的Vue本地路由，使用**Node.js反向代理**HTTP请求，类似于nginx，主要用于本地开发调试。

生产环境反向代理仍使用nginx等反向代理服务。

## 8. Node.js的特点是什么？适用哪些场景？

Node.js是基于Chrome V8引擎的JavaScript运行时，采用**事件驱动、非阻塞I/O模型**。

特点：
- **单线程事件循环**，通过异步回调处理高并发
- **非阻塞异步I/O**，适合I/O密集型场景
- 前后端统一使用JavaScript，降低切换成本
- npm拥有庞大的开源生态系统

适用场景：
- Web API / RESTful服务
- 实时应用（聊天、游戏、协同编辑）
- 前端工具链（webpack、vite、rollup等）
- BFF层（Backend For Frontend）

不适用场景：**CPU密集型任务**（大量计算会阻塞事件循环，应使用Worker线程或迁至后端语言）
