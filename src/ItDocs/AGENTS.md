## 7. 技术选型类答题示范

# 1. MySQL 和 PostgreSQL 怎么选？

**核心回答：**
MySQL 主打**轻量、高并发、易用、生态成熟**；PG 主打**复杂查询、强事务、GIS、JSON 支持、企业级特性**。

**核心区别：**

1. MySQL：读写高并发强、互联网生态无敌、运维简单、分库分表成熟
2. PG：复杂 SQL 强、多表join强、支持 GIS/全文检索、数据类型更丰富
3. MySQL 适合海量简单业务；PG 适合复杂查询、报表、地理信息、高完整性场景

**适用场景：**
MySQL 适用：电商、订单、用户中心、高并发交易、微服务通用业务
PG 适用：GIS 地图、物流轨迹、复杂报表、多表关联、金融强一致、非结构化数据

**实战选型：**
互联网公司首选 MySQL；需要 GIS、复杂查询、强数据完整性选 PG。

**一句话总结：**
高并发选 MySQL，复杂查询/GIS 选 PG。

# 2. MySQL 和 Elasticsearch 区别与选型

**核心回答：**
MySQL 是**关系型事务数据库**，ES 是**分布式搜索引擎**，擅长海量数据检索、聚合、模糊匹配。

**核心区别：**

1. MySQL：支持事务、强一致、适合精确增删改查
2. ES：非实时、搜索极快、模糊匹配/分词/聚合极强、不支持强事务
3. MySQL 适合业务数据；ES 适合搜索、日志、大数据量检索

**适用场景：**
MySQL：核心业务、订单、用户、支付、库存
ES：商品搜索、日志分析、全文检索、推荐、大屏聚合

**实战选型：**
核心业务必须用 MySQL；搜索、日志、大数据量检索用 ES。
常见架构：MySQL 存数据，ES 同步数据做检索。

**一句话总结：**
业务事务用 MySQL，检索搜索用 ES。
